## 什么是引导扇区

计算机上电启动后，首先运行主板 BIOS 程序， BIOS 并不知道如何加载操作系统，所以 BIOS 把**加载操作系统**的任务交给**引导扇区**。 因此，引导扇区必须放在一个已知的标准位置，这个位置就是磁盘的第一个扇区(cylinder 0, head 0, sector 0) 该扇区一共占用 512 字节。

为了确保 “磁盘是可引导的” BIOS 会检查扇区的第 511 和 512 字节是不是 `0xAA55`.

一个最简单的引导扇区：

```assembly
e9 fd ff 00 00 00 00 00 00 00 00 00 00 00 00 00
00 00 00 00 00 00 00 00 00 00 00 00 00 00 00 00
[ 29 more lines with sixteen zero-bytes each ]
00 00 00 00 00 00 00 00 00 00 00 00 00 00 55 aa
```

你看到的基本上都是 0，最后以 16 位的 `0xAA55` 结束。(注意字节顺序， x86 是小端结构)。开头的三字节表示计算机进入一个无限循环。

尝试编写一个最简单的引导扇区：

```assembly
; Infinite loop (e9 fd ff)
loop:
    jmp loop 

; Fill with 510 zeros minus the size of the previous code
times 510-($-$$) db 0
; Magic number
dw 0xaa55 
```

编译 `nasm -f bin boot_sect_simple.asm -o boot_sect_simple.bin` 然后尝试运行一下ta :  ` qemu boot_sect_simple.bin `

失败了 `qemu` 不是正确的命令。完整的命令名是 `qemu-system-x86_64` 所以我们需要使用  :  ` qemu-system-x86_64 boot_sect_simple.bin ` 
还是有一些问题 最好使用下面的命令：
` qemu-system-x86_64 -drive format=raw,file=boot_sect_simple.bin -nographic -m 32M `

- ✅ 使用 `-drive format=raw` 明确指定原始格式（消除警告）
- ✅ 使用 `-nographic` 以文本模式运行
- ✅ 使用 `-m 32M` 分配内存

![image-20260323161002537](/Users/xiatian/Library/Application Support/typora-user-images/image-20260323161002537.png)

最后我们成功运行了这个引导扇区 不过它实际上是让计算机进入了无限循环 

如果要退出 QEMU，按 `Ctrl+A` 然后输入 `x`（或 `Ctrl+C`）。

为了方便 我们可以创建一个脚本 `run_boot.sh`：

```shell
#!/bin/bash
qemu-system-x86_64 -drive format=raw,file=boot_sect_simple.bin -nographic -m 32M
```

这样只需要运行下面俩个命令即可：

```shell
chmod +x run_boot.sh # 这个命令是什么意思呢 实际上是给予权限
./run_boot.sh
```

## 引导打印

我们试着对之前的无限循环引导扇区做一些改进 让他可以打印东西在屏幕上。我们可以通过发出中断来完成这个功能。

在这个例子中我们将会向寄存器 `al` (`ax` 的低端部分) 写入 “hello” 单词的每个字符，把 `0x0e` 写入 `ah` (`ax` 的高端部分) 然后发出中断 `0x10`，这是视频服务的通用中断。

`0x0e` 写入 `ah` 告诉视频中断我们想要以 tty 模式显示 `al` 寄存器中的内容。

我们只会设置 tty 模式一次尽管实际上我们不能保证 `ah` 中的内容是不变的。在我们的程序休眠的时候，其他的进程可能仍然在 CPU 上执行，也许它们没有正确的清空 `ah` 甚至遗留一些垃圾数据在里面。

在这个例子中，我们不需要考虑这个问题，因为我们的引导扇代码是唯一能够在 CPU 上执行的程序。

我们的新引导扇区代码看起来是这个样子：

```assembly
mov ah, 0x0e ; tty mode
mov al, 'H'
int 0x10
mov al, 'e'
int 0x10
mov al, 'l'
int 0x10
int 0x10 ; 'l' is still on al, remember?
mov al, 'o'
int 0x10

jmp $ ; jump to current address = infinite loop

; padding and magic number
times 510 - ($-$$) db 0
dw 0xaa55 
```

你可以使用 `xxd file.bin` 命令检查二进制数据

当然你已经知道我们的编译和运行步骤 为了方便 我把`run_boot.sh`改了  (这样的话可以直接执行`./run_boot.sh`) ：

```shell
#!/bin/bash
nasm -f bin print_bss.asm -o print_bss.bin
qemu-system-x86_64 -drive format=raw,file=print_bss.bin -nographic -m 32M
```

现在你的启动引导程序会打印 'Hello' 然后停止在无限循环中。

### 什么是中断

在计算机的世界里，**中断（Interrupt）** 就像是生活中的“闹钟”或“插队请求”。

想象你正在专心致志地看书（CPU 正在执行程序），突然电话响了（产生了一个硬件信号）。你会放下书，去接电话（处理中断），接完电话后，你会回到刚才看书的那一页继续读（恢复执行原程序）。

如果没有中断，CPU 就得像个强迫症患者一样，不停地问每一个设备：“你有消息吗？你有消息吗？”，这在技术上被称为**轮询（Polling）**，非常浪费资源。

#### 1. 中断的核心分类

中断主要分为两大类，在这里引导扇区中主要涉及的是第二种：

- **硬件中断（External/Hardware Interrupts）：** 由外部硬件触发。比如你按下了键盘、移动了鼠标，或者硬盘数据读写完成了。
- **软件中断（Internal/Software Interrupts）：** 由程序代码主动触发。比如你在汇编中写的 `int 0x10`。这本质上是程序在**主动请求操作系统或 BIOS 的服务**。

#### 2. 中断的工作流程 (以 `int 0x10` 为例)

当你执行到 `int 0x10` 这行代码时，CPU 会经历以下“心理活动”：

1. **暂停：** 停下当前的活，把当前运行的位置（寄存器信息）存到内存的“栈”里，防止一会儿找不到回家的路。
2. **查找：** CPU 会去内存最开始的地方查找**中断向量表（IVT）**。这张表就像一份“拨号单”，告诉你 `0x10` 这个号码对应哪一段处理程序的地址。
3. **跳转：** CPU 跳到 BIOS 预设好的那段专门负责“视频输出”的代码去运行。
4. **执行：** BIOS 检查你的 `ah`（是 `0x0e` 吗？）和 `al`（是要打什么字？），然后驱动显卡在屏幕上画出像素。
5. **返回：** 执行完后，BIOS 发出一个 `iret` 指令，CPU 就像收到“收工”信号一样，从栈里取出刚才存的位置，跳回到你的引导代码继续跑。

#### 3. 为什么在引导扇区里它这么重要？

因为在引导扇区阶段，**你没有任何“库”可以使用**。

- 没有 `stdio.h`。
- 没有 `printf` 函数。
- 没有操作系统。

你唯一能依仗的“工具箱”就是 BIOS 固化在主板里的这些中断服务。`int 0x10` 是你跟显示器沟通的**唯一桥梁**。如果不通过中断，你就得自己去操作显卡的寄存器、自己去显存里写像素点，那工作量会指数级增长。

### 什么是视频

简单来说，这里的**“视频”指的是“显示输出”**。在计算机刚刚启动的实模式（Real Mode）下，你需要一种方式告诉硬件在屏幕上画出字符，而 `int 0x10` 就是 BIOS 留给你的“简易显卡驱动”。

#### 1. 什么是视频服务 (BIOS Video Services)？

在引导阶段，CPU 运行在极其原始的状态。你没有窗口系统，甚至没有现成的 `printf` 函数。为了让开发者能轻松地在屏幕上显示文字，主板上的 **BIOS** 内置了一组预设好的程序。

- **中断信号 `int 0x10`：** 这是一个指令，相当于你敲了敲 CPU 的门说：“嘿，帮我处理一下跟屏幕显示有关的事情。”
- **功能号 `ah = 0x0e`：** 这是一个具体的暗号。`0x0e` 代表“电传打字机模式”（Teletype Output）。它的作用是：在光标当前位置打印一个字符，并自动把光标往后挪一格。

#### 2. 寄存器是如何配合的？

这段话描述的是汇编语言中操作 BIOS 中断的标准流程。你可以把它想象成给一个函数传参：

- **`al` (低 8 位):** 存放你要显示的**字符**（比如 'h' 的 ASCII 码）。
- **`ah` (高 8 位):** 存放**功能代码**（`0x0e` 告诉 BIOS 你要打印字符）。
- **执行 `int 0x10`:** 触发动作。

#### 3. 为什么叫“通用中断”？

因为 `int 0x10` 不仅能打字，它还能做很多事，只要你改变 `ah` 的值：

- 设置屏幕分辨率（图形模式 vs 文本模式）。
- 改变背景颜色和文字颜色。
- 移动光标的位置。
- 读取光标所在位置的字符。

> **底层逻辑：** 当你发出 `int 0x10` 时，CPU 会暂停当前的工作，去查询“中断向量表”，找到 BIOS 存储在 ROM 中的显示处理程序代码，执行完打印动作后，再跳回你的引导代码继续运行。

#### 举个直观的例子

如果你想在屏幕上打一个字母 'A'，你的汇编代码看起来就像这样：

代码段：

```assembly
mov al, 'A'    ; 把字符 A 放到 al
mov ah, 0x0e   ; 告诉 BIOS 我要用电传打字模式
int 0x10       ; 呼叫视频服务
```
